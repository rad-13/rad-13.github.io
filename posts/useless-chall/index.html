INS'hAck 2019 - useless-chall

useless-chall è la penultima sfida della sezione reverse engineering della INS’hAck 2019 CTF. Benchè la sfida sia riportata nella sezione hash la maggior difficoltà trovata è stata nel bypassare le vare funzioni di hash piuttosto che capirle. Il programma è stato reversatro tramite il tool Ghidra.

Il binario utilizzato è il seguente: Download

La descrizione della sfida riporta:

One of my friend sent me this binary and asked me if it was secure enough to act as the auth page of his highly secure password manager.

NOTA: The flag is of the form INSA{…} and contains letters, numbers and symbols. And way too many characters to be able to bruteforce it :)

This binary accepts more than one valid input, if you found one.

Ci viene quindi comunicato che ci sono più soluzioni a tale sfida, ma che il briteforcing non è una soluzione.

È possibile entrare in tale condizione dato il segno del valore di hash, in pratica sta controllando in modo abbastanza malato che il numero non sia dispari: $h^2 + 1 = \frac{(h^2 + 1) * 7}{7}$, ciò è vero a meno di arrotondamenti (che potremmo avere dato il casting).

Il problema principale della sfida sorge invece nella condizione imposta da hashFun2, ovvero:

hash2_t = hashFun2(str);
if ((int)hash2_t == 0x3c5ae529){...}

Ci viene chiesto di trovare una collisione per la funzione di hash tale che il valore di hash restituito sia pari a 0x3c5ae529.

La consegna del problema prende ora più senso, esistono più soluzioni poichè c’è possibilità di collisione nella funzione di hash. Decompilando l’impllentazione di hashFun2 si ricava il seguente codice:

unsigned long hashFun2(char *str)
{
  size_t sLen;
  uint sum;
  int i;

  sLen = strlen(str);
  sum = 0;
  i = 0;
  while (i < (int)sLen) {
    sum = sum + ((int)str[(long)i] << ((char)i & 0x1f));
    i = i + 1;
  }
  return (unsigned long)sum;
}

È una sommatoria, più precisamente imponendo il vincolo che la stringa in input è di 41 caratteri abbiamo che la funzione appena vista implementa la seguente formula matematica: $$ \begin{align} hash = \sum_{i=0}^{41}n_i \cdot 2^{i \% 31} \end{align} $$ Tale funzione di hash non è sicura, risulta possibile ottenere una collisione verso il valore richiesto.

Per farlo si analizza la formula appena data.

La distribuzione dell’esponente ci mostra che esso si incrementa man mano che $i$ aumenta, in pratica ciò implica che il contributo di $n_i$ aumenta man mano che la $i$ si avvicina a 31, per poi ricominciare da 0 man mano si allontana.

In pratica ciò ci sta dicendo che se vogliamo forzare dei grossi cambiamenti nell’output dovremo farlo con le prime cifre a sinistra di 31. Inoltre se vogliamo cambiare un singolo byte nell’output basterà aumentare o diminuire il 31-esimo byte nell’input, poichè $n_i \cdot 2^{31 \% 31} = n_i \cdot 2^{0} = 1$. L’idea è quindi quella di lasciar perdere le ultime $(41-31) = 10$ cifre e lavorare su quelle più significative.

L’algoritmo parte inizializzando un vettore di ‘A’ (41 in intero).

Fatto ciò si sceglie la 31-esima cifra e si incrementa il valore fino a che l’hash ottenuto dalla stringa in esame non supera il valore di 0x3c5ae529, a questo punto si decrementa la cifra attuale e ci si sposta di una lettera a destra, così che al prossimo ciclo si potrà raffinare la sequenza verso l’hash voluto.

L’algoritmo tende quindi alla soluzione ottima, non è detto che la raggiunga (dipende dalle condizioni iniziali), se si parte dal carattere ‘A’ come in esempio si ottiene la flag.

Il codice completo è il seguente:

from numpy import int32

def toStr(l):
    return ''.join([chr(x) for x in l])

def hashFun2(str):
    sLen = len(str)

    sum = 0
    i = 0
    while(i<int(sLen)):
        sum = int32(sum + ((ord(str[i])) << (i & 0x1f)))

        i = i + 1

    return int32(sum)

hash2 = 0x3c5ae529
blist = [ord('A')] * 41

for i in range(31,0,-1):
    for j in range(1,126-ord('A')):
        h2 = hashFun2(toStr(blist))

        if(h2 < hash2):
            blist[i] = blist[i] + 1
        else:
            blist[i] = blist[i] - 1
            break

blist[32] = blist[32] + 1
h2 = hashFun2(toStr(blist))

print(hex(h2))
print(blist)
print(toStr(blist))
